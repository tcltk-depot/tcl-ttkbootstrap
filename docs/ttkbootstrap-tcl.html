Options: -text, -variable, -bootstyle, -command, -shape (round|square)
Note: SVGToggleSwitch is a frame-based composite widget, not a ttk button, so it does not respond to $w invoke. Set its -variable to change state programmatically (the switch redraws and -command fires); read that variable to get its state. Users toggle it by clicking.

Gotchas & Common Pitfalls

Input widgets wrap a child entry
SVGEntry, SVGCombobox, SVGSpinbox, SVGSearchBar and SVGFormField place the editable control in a child path. Call get/insert/delete on the child, not the outer path.
# 1.5.0+: use the accessors (recommended)
ttkbootstrap::SVGEntry::set .e "hello"
ttkbootstrap::SVGEntry::get .e
ttkbootstrap::SVGEntry::clear .e
# (same get/set/clear for SVGCombobox, SVGSpinbox, SVGSearchBar, SVGFormField)

# or reach into the child path directly:
SVGEntry .e      -> .e.ent
SVGCombobox .cb  -> .cb.cb
SVGSpinbox .sp   -> .sp.sp
SVGSearchBar .sb -> .sb.ent
SVGFormField .ff -> .ff.ent.ent
# or pass -textvariable and use that variable
Composite widgets are not ttk widgets
SVG* widgets are built from frame/label/canvas. They do not support $w invoke, $w state, or ttk-style cget. Use options, -variable, and Widget::method subcommands.
Errors are raised, not returned
Bad theme names, color keys, and -bootstyle values raise descriptive errors. Wrap calls in catch when passing user-supplied values.
Widget paths must be unique
Re-creating a widget at an existing path raises the standard Tk "window name already exists" error. Destroy the old one first.
Numeric options are not range-checked
Reversed scale ranges, a zero meter total, or fractional ratings are accepted without error but may render oddly. Validate user input yourself.
Test suite needs a UTF-8 locale
Run with LANG=en_US.UTF-8 or the box-drawing characters in the test output raise an encoding error.

EXAMPLES

Minimal App

source ttkbootstrap.tcl
ttkbootstrap::Window -themename litera -title "Hello" -size {400 300}
ttk::button .btn -text "Click" -style "primary.TButton"     -command {puts "Hello!"}
pack .btn -pady 20

Dashboard with Shadow Cards

source ttkbootstrap.tcl
ttkbootstrap::Window -themename darkly -title "Dashboard" -size {800 500}

set main [ttk::frame .main]
pack $main -fill both -expand 1 -padx 16 -pady 16

foreach {title value bs} {
    "Active Tasks" "12" primary
    "Completed" "48" success
    "Overdue" "3" danger
} {
    set c [ttkbootstrap::SVGShadowCard $main.c[incr i]         -title $title -bootstyle $bs -shadow 10 -width 200 -height 120]
    set body [ttkbootstrap::SVGShadowCard::body $c]
    ttk::label $body.v -text $value -font {TkDefaultFont 24 bold}
    pack $body.v -pady 8
    pack $c -side left -padx 8 -fill both -expand 1
}

Data Entry Form

source ttkbootstrap.tcl
ttkbootstrap::Window -themename minty -title "Registration" -size {500 400}

foreach {lbl var} {Name ::name Email ::email Phone ::phone} {
    set f [ttk::frame .f[incr i]]
    ttk::label $f.l -text "$lbl:" -width 8
    ttkbootstrap::SVGEntry $f.e -bootstyle primary         -textvariable $var -width 30
    pack $f.l -side left
    pack $f.e -side left -padx 8
    pack $f -pady 4 -padx 16 -anchor w
}

ttkbootstrap::PillButton .submit     -text "Register" -bootstyle success     -command {puts "Name: $::name, Email: $::email"}
pack .submit -pady 16
